## What you can do

- Register users and request humanness checks.
- Retrieve humanness scores, unique human IDs, and other verification signals.
- Receive webhook events when verification completes, is revoked, or cannot be fulfilled.

## How it works

1. You register a user and specify what data you need (e.g. humanness score, unique human ID, phone verification).
1. The API returns an `invite_url` that you present to the user.
1. The user completes the verification flow on their mobile device.
1. You poll for status or listen for a webhook to know when verification is done.
1. You retrieve and decrypt the verification response to get the results.

## Quick start at a glance

1. Create a service account and API key.
1. Register a user with a data request type.
1. Present the invite URL to the user.
1. Check connection status or listen for webhooks.
1. Decrypt the response to read verification results.

Continue to `quickstart` for the step-by-step flow.
